## 1. 使用规则

1. 本台账用于记录 ATR 改造执行状态，不替代设计文档和执行计划。
2. 每个步骤状态变化时，必须同步更新“执行总表”和“审核记录”。
3. 每个审核发现的问题，必须进入“缺陷台账”。
4. 新增设计判断、兼容策略变化或范围调整时，必须进入“决策台账”。
5. 发现长期阻塞、跨仓依赖或回归风险时，必须进入“风险台账”。
6. 缺陷状态使用：`open`、`fixing`、`re-review`、`closed`。
7. 决策状态使用：`active`、`superseded`、`blocked`。
8. 风险状态使用：`open`、`mitigating`、`closed`。
9. `Explore 子代理` 负责自动化独立审查；步骤进入 `accepted` 或 `frozen` 前，仍应由项目负责人进行最终人工确认。
